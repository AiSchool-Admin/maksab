## 2. أداة اكتشاف البائعين المحترفين

### الوصف
إضافة متصفح (Chrome Extension) تُستخدم يدوياً لتحليل صفحات البائعين على OLX
وجمع بياناتهم بشكل منظم للتواصل معهم لاحقاً.

### الملفات

```
scripts/acquisition/
├── olx-extension/
│   ├── manifest.json          # Chrome Extension manifest
│   ├── content.js             # يُحقن في صفحات OLX
│   ├── popup.html             # واجهة الإضافة
│   ├── popup.js               # منطق الواجهة
│   ├── background.js          # Service worker
│   └── styles.css             # تنسيق الواجهة
├── bulk-import.ts             # سكريبت الإدخال المجمع
├── outreach-templates.ts      # رسائل WhatsApp جاهزة
└── README.md                  # دليل الاستخدام
```

### كيف تعمل الإضافة

```
1. المستخدم (أنت) يفتح صفحة بائع على OLX
2. الإضافة تستخرج تلقائياً:
   - اسم البائع
   - رقم الهاتف (لو ظاهر)
   - عدد الإعلانات النشطة
   - الأقسام اللي بيعلن فيها
   - متوسط الأسعار
   - تاريخ الانضمام
   - تقييم البائع
3. البيانات تُحفظ في ملف JSON محلي
4. يمكن تصدير البيانات كـ CSV
5. لاحقاً: يتم إدخالها عبر Bulk Import Script
```

### معايير "البائع المحترف"

```typescript
interface SellerScore {
  // كل معيار من 10
  activeAdsCount: number;      // 10+ إعلانات = 10 نقاط
  accountAge: number;          // أكثر من سنة = 10 نقاط
  responseRate: number;        // نسبة الرد العالية = 10 نقاط
  adQuality: number;           // صور + وصف كامل = 10 نقاط
  categoryFocus: number;       // متخصص في قسم = 10 نقاط
  totalScore: number;          // المجموع من 50
  tier: 'platinum' | 'gold' | 'silver' | 'bronze';
}

// التصنيف:
// Platinum: 40-50 (أولوية قصوى — تواصل فوري)
// Gold:     30-39 (أولوية عالية — تواصل خلال أسبوع)
// Silver:   20-29 (أولوية متوسطة — تواصل خلال أسبوعين)
// Bronze:   10-19 (أولوية منخفضة — بعد استنفاد الأعلى)
```

### الاستخدام الآمن

> **مهم جداً:** هذه الأداة تعمل يدوياً — المستخدم يزور كل صفحة بنفسه.
> - لا يوجد scraping تلقائي أو crawling
> - لا يوجد تجاوز لـ rate limits
> - لا يتم جمع بيانات خاصة (فقط المعلومات العامة المعروضة)
> - الغرض: تسهيل النسخ اليدوي فقط

## 3. سكريبت الإدخال المجمع (Bulk Import)

### الوصف
سكريبت Node.js يأخذ ملف JSON/CSV يحتوي على بيانات بائعين مُجمّعة
ويُدخلها في قاعدة بيانات مكسب بشكل آمن ومنظم.

### ما يفعله السكريبت

```
1. يقرأ ملف البيانات (JSON أو CSV)
2. يتحقق من صحة البيانات (validation)
3. يتحقق من عدم التكرار (deduplication بالهاتف)
4. يُنشئ سجلات في جداول:
   - acquisition_leads: بيانات العميل المحتمل
   - acquisition_outreach: سجل محاولات التواصل
5. لا يُنشئ حسابات فعلية — فقط leads
6. يولّد تقرير بالنتائج
```

### تدفق البيانات

```
ملف CSV/JSON
    ↓
[Validation] → رفض البيانات الناقصة/الخاطئة
    ↓
[Deduplication] → تخطي الأرقام الموجودة بالفعل
    ↓
[Enrichment] → إضافة category mapping + seller score
    ↓
[Insert to DB] → acquisition_leads table
    ↓
[Report] → عدد المُضاف / المرفوض / المكرر
```

### صيغة ملف الإدخال

```json
{
  "sellers": [
    {
      "name": "محمد أحمد",
      "phone": "01012345678",
      "source": "olx",
      "source_profile_url": "https://olx.com.eg/profile/xxxxx",
      "categories": ["cars", "phones"],
      "active_ads_count": 15,
      "location": {
        "governorate": "القاهرة",
        "city": "مدينة نصر"
      },
      "seller_score": 42,
      "seller_tier": "platinum",
      "notes": "تاجر سيارات — 15 إعلان نشط — يرد بسرعة"
    }
  ]
}
```

## 5. تتبع الاستحواذ (Acquisition Tracking)

### جدول acquisition_leads

```sql
CREATE TABLE acquisition_leads (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  phone VARCHAR(11) NOT NULL UNIQUE,
  name VARCHAR(100),
  source VARCHAR(50) NOT NULL,          -- 'olx', 'facebook', 'marketplace', 'manual', 'referral'
  source_profile_url TEXT,
  categories TEXT[],                     -- الأقسام اللي بيعلن فيها
  active_ads_count INTEGER DEFAULT 0,
  seller_score INTEGER DEFAULT 0,
  seller_tier VARCHAR(20),              -- 'platinum', 'gold', 'silver', 'bronze'
  governorate VARCHAR(50),
  city VARCHAR(100),
  notes TEXT,

  -- Status tracking
  status VARCHAR(30) DEFAULT 'new',     -- 'new', 'contacted', 'interested', 'registered', 'active_seller', 'declined', 'blacklist'
  contacted_at TIMESTAMPTZ,
  registered_at TIMESTAMPTZ,
  first_ad_at TIMESTAMPTZ,
  registered_user_id UUID,

  -- Metadata
  imported_by VARCHAR(100),             -- اسم الشخص اللي أضاف الـ lead
  batch_id VARCHAR(50),                 -- رقم الدُفعة (للتتبع)
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

### جدول acquisition_outreach

```sql
CREATE TABLE acquisition_outreach (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  lead_id UUID REFERENCES acquisition_leads(id),
  channel VARCHAR(20) NOT NULL,         -- 'whatsapp', 'sms', 'call', 'facebook_dm'
  template_id VARCHAR(50),              -- أي قالب تم استخدامه
  message_preview TEXT,
  status VARCHAR(20) DEFAULT 'sent',    -- 'sent', 'delivered', 'read', 'replied', 'opted_out'
  sent_at TIMESTAMPTZ DEFAULT NOW(),
  replied_at TIMESTAMPTZ,
  response_summary TEXT,
  sent_by VARCHAR(100)                  -- اسم المُرسل
);
```
