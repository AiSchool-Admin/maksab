## 🎯 المشكلة

```
لو حصدنا 5,000 إعلان موبايلات (عرض) و 0 مشترين موبايلات (طلب):
  → البائعين يلاقوا مفيش مشترين → يسيبوا مكسب!

لو حصدنا 5,000 مشتري موبايلات (طلب) و 0 إعلانات (عرض):
  → المشترين يلاقوا مفيش إعلانات → يسيبوا مكسب!

الحل: تناسب ذكي — كل فئة × محافظة عندها نسبة عرض/طلب مثالية
```

### الطبقة 2: توزيع الموارد (Resource Allocation)

```
═══ قاعدة 70/20/10 ═══

70% من موارد الحصاد → أعلى 5 (🔴):
  📱 موبايلات القاهرة + الإسكندرية + الجيزة
  🚗 سيارات القاهرة + الإسكندرية

20% من الموارد → المتوسط (🟠🟡):
  🏠 عقارات القاهرة + الإسكندرية
  💻 إلكترونيات القاهرة + الإسكندرية
  📱 موبايلات القليوبية

10% من الموارد → الباقي (🟢):
  باقي المحافظات + باقي الفئات

═══ ترجمة لـ harvest intervals ═══

🔴 أعلى 5:
  AHE (بائعين): كل 30 دقيقة | 5 صفحات | detail fetch 50
  BHE (مشترين): Paste&Parse مرتين/يوم + تعليقات دوبيزل مع كل حصادة

🟠🟡 متوسط:
  AHE: كل 60 دقيقة | 3 صفحات | detail fetch 20
  BHE: Paste&Parse مرة/يوم + تعليقات دوبيزل

🟢 الباقي:
  AHE: كل 360 دقيقة | 2 صفحات | detail fetch 10
  BHE: Paste&Parse مرة/أسبوع
```

### الطبقة 3: التوازن الديناميكي (Dynamic Balancing)

```
═══ المفهوم: النظام يراقب ويعدّل تلقائياً ═══

كل فئة × محافظة عندها:
  supply_count = عدد الإعلانات النشطة (من AHE)
  demand_count = عدد المشترين النشطين (من BHE)
  ratio = supply_count / demand_count
  target_ratio = النسبة المثالية للفئة

لو ratio > target_ratio (عرض أكتر من اللازم):
  → قلل حصاد البائعين
  → زوّد حصاد المشترين
  → "عندنا بائعين كتير ومحتاجين مشترين أكتر"

لو ratio < target_ratio (طلب أكتر من اللازم):
  → زوّد حصاد البائعين  
  → قلل حصاد المشترين
  → "عندنا مشترين كتير ومحتاجين بائعين أكتر"

═══ مثال عملي ═══

موبايلات القاهرة:
  supply = 500 إعلان نشط
  demand = 800 مشتري نشط
  ratio = 0.625
  target = 0.33 (1:3)
  ratio > target → عندنا عرض أكتر من اللازم!
  
  الإجراء:
  → AHE: قلل interval من 30 → 60 دقيقة (توفير موارد)
  → BHE: زوّد حصاد المشترين (Paste&Parse 3 مرات/يوم)
  → ركّز التواصل: ابعت للمشترين "لقينا اللي بتدوّر عليه!"

سيارات الإسكندرية:
  supply = 50 إعلان
  demand = 10 مشترين
  ratio = 5.0
  target = 0.20 (1:5)
  ratio >> target → عرض كتير جداً ومفيش طلب!
  
  الإجراء:
  → AHE: أوقف مؤقتاً (عندنا كفاية)
  → BHE: ركّز بالكامل على جلب مشترين سيارات
  → التسويق: "بتدوّر على سيارة في الإسكندرية؟ 50 سيارة على مكسب!"
```

## 🤖 التنفيذ التقني: Dynamic Balancer

```sql
-- ════════════════════════════════════════════
-- جدول التوازن (يُحدّث تلقائياً كل ساعة)
-- ════════════════════════════════════════════
CREATE TABLE market_balance (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  
  category TEXT NOT NULL,
  governorate TEXT,  -- NULL = كل المحافظات
  
  -- العرض
  active_listings INTEGER DEFAULT 0,
  new_listings_today INTEGER DEFAULT 0,
  sellers_with_phone INTEGER DEFAULT 0,
  
  -- الطلب
  active_buyers INTEGER DEFAULT 0,
  new_buyers_today INTEGER DEFAULT 0,
  buy_requests_active INTEGER DEFAULT 0,
  
  -- النسبة
  supply_demand_ratio NUMERIC(6,2),
  target_ratio NUMERIC(6,2),
  balance_status TEXT, -- 'balanced' | 'oversupply' | 'undersupply'
  
  -- الإجراء المقترح
  recommended_action TEXT,
    -- 'increase_buyer_harvest'
    -- 'increase_seller_harvest'
    -- 'maintain'
    -- 'reduce_seller_harvest'
  
  -- Harvest Settings التلقائي
  ahe_interval_minutes INTEGER,
  ahe_max_pages INTEGER,
  bhe_paste_frequency TEXT, -- 'thrice_daily' | 'twice_daily' | 'daily' | 'weekly'
  
  updated_at TIMESTAMPTZ DEFAULT now(),
  
  UNIQUE(category, governorate)
);
```

```javascript
// ═══ Cron: Balance Calculator — كل ساعة ═══

async function calculateMarketBalance() {
  const categories = ['phones', 'vehicles', 'properties', 'electronics', 'furniture'];
  const governorates = ['cairo', 'alexandria', 'giza', null]; // null = overall
  
  const targetRatios = {
    phones: 0.33,      // 1 بائع : 3 مشترين
    vehicles: 0.20,    // 1 : 5
    properties: 0.10,  // 1 : 10
    electronics: 0.33, // 1 : 3
    furniture: 0.25,   // 1 : 4
  };
  
  for (const cat of categories) {
    for (const gov of governorates) {
      // عدد الإعلانات النشطة
      let listingsQuery = supabase
        .from('ahe_listings')
        .select('id', { count: 'exact', head: true })
        .eq('maksab_category', cat)
        .eq('is_duplicate', false);
      if (gov) listingsQuery = listingsQuery.eq('governorate', gov);
      const { count: supply } = await listingsQuery;
      
      // عدد المشترين النشطين
      let buyersQuery = supabase
        .from('bhe_buyers')
        .select('id', { count: 'exact', head: true })
        .eq('category', cat)
        .eq('is_duplicate', false)
        .in('pipeline_status', ['discovered', 'phone_found', 'matched']);
      if (gov) buyersQuery = buyersQuery.eq('governorate', gov);
      const { count: demand } = await buyersQuery;
      
      const ratio = demand > 0 ? supply / demand : supply > 0 ? 999 : 0;
      const target = targetRatios[cat] || 0.33;
      
      let status, action, aheInterval, ahePages, bheFreq;
      
      if (ratio > target * 3) {
        // عرض زائد جداً
        status = 'oversupply';
        action = 'increase_buyer_harvest';
        aheInterval = 120; // بطّئ حصاد البائعين
        ahePages = 2;
        bheFreq = 'thrice_daily'; // كثّف حصاد المشترين
      } else if (ratio > target * 1.5) {
        // عرض أكتر شوية
        status = 'oversupply';
        action = 'increase_buyer_harvest';
        aheInterval = 60;
        ahePages = 3;
        bheFreq = 'twice_daily';
      } else if (ratio < target * 0.5) {
        // طلب أكتر بكتير
        status = 'undersupply';
        action = 'increase_seller_harvest';
        aheInterval = 30; // سرّع حصاد البائعين
        ahePages = 5;
        bheFreq = 'daily';
      } else {
        // متوازن
        status = 'balanced';
        action = 'maintain';
        aheInterval = 60;
        ahePages = 3;
        bheFreq = 'daily';
      }
      
      await supabase.from('market_balance').upsert({
        category: cat,
        governorate: gov,
        active_listings: supply,
        active_buyers: demand,
        supply_demand_ratio: ratio,
        target_ratio: target,
        balance_status: status,
        recommended_action: action,
        ahe_interval_minutes: aheInterval,
        ahe_max_pages: ahePages,
        bhe_paste_frequency: bheFreq,
        updated_at: new Date().toISOString()
      }, { onConflict: 'category,governorate' });
    }
  }
}

// ═══ تطبيق التوازن على AHE تلقائياً ═══

async function applyBalanceToScopes() {
  const { data: balances } = await supabase
    .from('market_balance')
    .select('*')
    .not('governorate', 'is', null);
  
  for (const balance of balances || []) {
    // حدّث scope الـ AHE المقابل
    await supabase
      .from('ahe_scopes')
      .update({
        harvest_interval_minutes: balance.ahe_interval_minutes,
        max_pages_per_harvest: balance.ahe_max_pages
      })
      .eq('maksab_category', balance.category)
      .eq('governorate', balance.governorate);
  }
}
```
