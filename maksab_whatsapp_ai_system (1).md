## 🧠 المفهوم: 5 طبقات ذكية

```
الطبقة 1 — الاكتشاف (تلقائي 100%):
  محرك الحصاد يجمع الإعلانات + الأرقام + الأسماء
  → يصنّف: فرد | تاجر | حوت 🐋
  → يحسب الأولوية: Whale Score 0-100
  = مكتمل وشغال ✅

الطبقة 2 — التواصل المبدئي (تلقائي 100%):
  رسالة واتساب ترحيبية مخصصة حسب:
    الفئة (موبايلات/سيارات/عقارات)
    نوع المعلن (فرد/تاجر/حوت)
    المحافظة واللهجة
  = المطلوب بناؤه

الطبقة 3 — المحادثة الذكية (AI 100%):
  Claude AI يرد على استفسارات المعلن تلقائياً
  يفهم: مهتم | مش مهتم | عنده أسئلة | عايز يعرف أكتر
  يرد بالعامية المصرية — زي موظف خدمة عملاء محترف
  = المطلوب بناؤه

الطبقة 4 — التسجيل والنشر (تلقائي 100%):
  لما المعلن يوافق:
  → يبعتله رابط تسجيل مخصص (OTP)
  → لما يسجل: "لقينا إعلاناتك! تحب ننقلهم؟"
  → بضغطة واحدة: إعلاناته تتنشر على مكسب
  = المطلوب بناؤه

الطبقة 5 — المتابعة والنمو (تلقائي 100%):
  بعد التسجيل:
  → تهنئة + نصائح لتحسين الإعلانات
  → بعد أسبوع: "إعلاناتك حققت XX مشاهدة!"
  → بعد شهر: "تحب تجدد؟" + عروض ترقية
  → تنبيهات: "حد مهتم بإعلانك!" 
  = المطلوب بناؤه
```

## 🤖 المحادثة الذكية (AI Conversation Engine)

### كيف يعمل:

```
كل رسالة واردة من واتساب:
  
  1. Webhook يستقبل الرسالة
  2. يحدد: من أي عميل؟ في أي مرحلة؟
  3. يبعت لـ Claude API مع السياق:
     - تاريخ المحادثة
     - بيانات العميل (اسم + فئة + إعلانات + مرحلة)
     - التعليمات (System Prompt)
  4. Claude يرد
  5. الرد يتبعت عبر واتساب API
  6. يُحدّث حالة العميل في CRM
```

### System Prompt لـ Claude AI:

```
أنت "سارة" — موظفة خدمة عملاء في تطبيق مكسب.
مكسب هو أول تطبيق مصري للبيع والشراء والمزادات والتبادل.

═══ شخصيتك ═══
- بتتكلمي بالعامية المصرية (مش فصحى)
- ودودة ومحترفة — زي موظفة بنك محترمة
- بتفهمي اللهجات المصرية المختلفة
- مش بتضغطي على حد — لو حد مش مهتم بتحترمي رأيه
- بتستخدمي إيموجي باعتدال (مش كتير)

═══ معلومات عن مكسب ═══
- أول تطبيق مصري يجمع: بيع مباشر + مزادات + تبادل/بارتر
- 12 فئة: موبايلات + سيارات + عقارات + إلكترونيات + ...
- العمولة طوعية 1% (مش إجبارية — ده الفرق عن دوبيزل)
- باقات التجار: Free (10 إعلانات) | Silver 199ج.م | Gold 499ج.م | Diamond 999ج.م
- مميزات: مزادات + تبادل + واتساب مدمج + تحليلات + صفحة متجر

═══ بيانات العميل الحالي ═══
الاسم: {{customer_name}}
الفئة: {{category}}
عدد إعلاناته: {{listings_count}}
المحافظة: {{governorate}}
نوعه: {{seller_type}} (فرد/تاجر/حوت)
المرحلة: {{lifecycle_stage}}
آخر رسائل: {{conversation_history}}

═══ مهمتك ═══
1. لو العميل جديد (lead): رحبي بيه واشرحيله مميزات مكسب
2. لو مهتم: ساعديه يسجل واشرحيله إزاي ينقل إعلاناته
3. لو عنده أسئلة: جاوبيه بوضوح
4. لو مش مهتم: احترمي رأيه — قولي "لو غيرت رأيك في أي وقت كلمنا"
5. لو عنده مشكلة تقنية: ساعديه أو حوليه لفريق الدعم
6. لو عايز يتكلم مع إنسان: قولي "هوصلك بزميلي الآن"

═══ قواعد مهمة ═══
- لا تختلقي معلومات مش موجودة (أسعار / مميزات غير حقيقية)
- لا تعدي على 3 رسائل متتالية من غير رد (spam)
- لو الشخص قال "بطّل" أو "لا شكراً" → توقفي فوراً
- الردود قصيرة (2-4 سطور) — واتساب مش إيميل
- لا تذكري إنك AI أبداً — أنت "سارة من فريق مكسب"
```

## ⚙️ محركات النظام

### المحرك 1: Outreach Engine (إرسال تلقائي)

```
يعمل على Railway — كل 5 دقائق

GET /cron/outreach

الآلية:
  1. اجلب المعلنين الجدد اللي عندهم أرقام ولم يُتواصل معهم:
     SELECT s.* FROM ahe_sellers s
     LEFT JOIN wa_conversations wc ON wc.phone = s.phone
     WHERE s.phone IS NOT NULL
     AND s.pipeline_status IN ('phone_found', 'auto_queued')
     AND wc.id IS NULL
     ORDER BY s.whale_score DESC, s.priority_score DESC
     LIMIT 5
  
  2. لكل معلن:
     a. أنشئ wa_conversations
     b. حدد القالب المناسب (من wa_automation_rules)
     c. ابعت الرسالة (WhatsApp Business API أو Claude AI)
     d. حدّث: pipeline_status = 'contacted'
  
  3. اجلب الرسائل المجدولة:
     SELECT * FROM wa_conversations
     WHERE next_action_at <= now()
     AND status IN ('waiting', 'scheduled')
     LIMIT 10
  
  4. لكل محادثة مجدولة:
     a. نفذ الإجراء (إرسال رسالة/تذكير/إغلاق)
     b. حدّث الحالة

  حد أقصى: 20 رسالة/ساعة (WhatsApp rate limits)
```

### المحرك 2: Response Handler (استقبال الردود)

```
Webhook على Vercel:
POST /api/webhooks/whatsapp

الآلية:
  1. استقبل الرسالة من WhatsApp Webhook
  2. حدد المحادثة: wa_conversations WHERE phone = incoming_phone
  3. لو مفيش محادثة → محادثة جديدة (عميل بيبدأ هو)
  
  4. ابعت لـ Claude API:
     System: [System Prompt بتاع سارة]
     Messages: [تاريخ المحادثة + الرسالة الجديدة]
     
  5. Claude يرد + يصنف النية:
     {
       "reply": "أهلاً أحمد! مكسب فيها مميزات...",
       "intent": "positive",
       "action": "send_signup_link",
       "sentiment": "positive",
       "should_escalate": false
     }
  
  6. ابعت الرد عبر WhatsApp API
  7. حدّث المحادثة والـ CRM
  
  8. لو should_escalate:
     → أرسل إشعار للمدير
     → حوّل المحادثة لموظف بشري

### المحرك 3: Lifecycle Engine (دورة حياة العميل)

```
Cron يومي — الساعة 9 صباحاً (أفضل وقت للتواصل)

GET /cron/lifecycle

الآلية:
  لكل trigger في wa_automation_rules:
    
    'no_response_24h':
      SELECT * FROM wa_conversations
      WHERE last_message_direction = 'outbound'
      AND last_message_at < now() - INTERVAL '24 hours'
      AND messages_received = 0
      AND stage = 'initial_outreach'
      → ابعت متابعة
    
    'weekly_stats':
      SELECT * FROM crm_customers
      WHERE lifecycle_stage = 'activated'
      AND last_weekly_stats_at < now() - INTERVAL '7 days'
      → ابعت إحصائيات
    
    'listing_expiring':
      SELECT * FROM ahe_listings
      WHERE is_expired = false
      AND estimated_posted_at < now() - INTERVAL '25 days'
      → ابعت تنبيه
    
    'upgrade_eligible':
      SELECT * FROM crm_customers
      WHERE lifecycle_stage = 'activated'
      AND total_listings >= 15
      AND subscription_plan = 'free'
      → ابعت عرض ترقية
